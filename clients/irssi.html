Using Irssi to Access the Desert Bus Chat

Irssi is a popular command line IRC client amongst enthusiasts, but not the easiest for new users. It has several benefits, from being combined with other programs for enabling an always on IRC client regardless if you are logged into your machine, to being highly configurable and adaptable with plugins.

If you would like to use Irssi, here are some tips to get up and running with the Desert Bus Chat quickly.

  1. Starting Irssi:

    Assuming you've installed irssi...

    • On *nix machines, get to a terminal window and type irssi
    • On Mac, (someone with experience edit)
    • On Windows Machines, find wherever you extracted the program files to, and run irssi.cmd

    2. To add DesertBus to the irssi configuration file so you don't have to remember anything, you need to follow a 2-step process. The first step defines a network and sets the nick you are going to use on it. The second step defines the server details of the network, and attaches it to the network definition.

    Once the two steps are completed, when irssi starts again, it will automatically connect to the defined network and related server.

  2. Add a network:

    • /network add -nick YourNickname DesertBus

  3. Add the server:

    • /server add -ssl -auto -network DesertBus irc.desertbus.org 6697 your.email@example.com:secretpassword
      Important: use the email address and password of your donor account
      Note: -auto in this command tells Irssi to automatically connect to the Desert Bus chat on startup; if you would prefer not to do this, remove -auto
      Note: if you make a mistake /server list will list what you've done, and /server remove (server name) should remove incorrect entries

  4. Aliases:

    Optionally, add some handy aliases so that /bid and /ns etc. work as they should (the first is the most important)

    • /alias bid quote bid
    • /alias ns quote ns
    • /alias login quote login
    • /alias id quote id
    • /alias bidserv quote bidserv
    • /alias bs quote bs

  5. Cleanup the Chat:

    Optionally, to hide joins, quits, etc. on the chat (which gets very busy during Desert Bus)

    • /ignore -channels #desertbus * JOINS PARTS QUITS NICKS
    • /ignore -channels #desertbus ChanServ MODES

  6. Save your configuration:

    • /save

  7. Connect:

    • /connect DesertBus

You should now be in the Desert Bus chat; enjoy!

Using Irssi

The /help command

You can find out about the various commands in irssi by using /help to list the main commands, and /help irssiCommand.

Windows

Irrsi uses the concept of windows to keep track of where things are. Generally, window 1 holds all the network/server details and displays configuration commands and their output. Channel output, such as Desert Bus will generally be in windows 2 and onwards.

  • /window #

    You can use the above command to move between the windows. You can also, unless otherwise configured, use alt + left/right and alt + numkey.

  • /window close
  • /window move

Customising Irssi

Irssi can be customised by exploring the /set command. You can set such things as logging preferences, the time display, your default nick and username, and more.

  • /set timestamp_format

    Default is %H:%M, you can change to %H:%M:%S to display seconds, handy during bidding wars, and other configurations including days, months and years.

  • /set autolog ON

    Turns autolog functionality on, keeps a record of all activity you see in your channels

  • /set scrollback_time

    Set how much scrollback you want Irssi to keep, default is 1day

Hilighting

You can hilight text, users, etc in Irssi with the /hilight command.

  • /hilight word

    This will hilight word whenever it appears in a message

  • /hilight -mask nick!*@*

    This will hilight all messages from the user nick

  • -nick, -word, and -line

    You can modify how hilight behaves on each command with the above variables. For instance if you want only the word you're looking for hilighted in a message, use /hilight -word x

The Always On Client

Particularly on *nix machines, when combined with programs like screen or tmux, and remote login with ssh, your irssi session can become your always on client, accessible from anywhere with the right network settings, keeping you always connected to Desert Bus, and Loading Ready Run's twitch chat as well.
